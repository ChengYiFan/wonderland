## DRY原则
Don't Repeat Yourself.
能计算就不要存储，能无状态尽量无状态。
为了降低系统复杂度。如果计算得到的状态单独存储，固然性能会好一点，但是带来了维护一致性的复杂度。每当条件状态发生改变，都需要维护计算状态保持一致。 尽量将所有状态都放到一个地方管理，可以降低系统的复杂度。无状态的组件更容易测试和调试。
